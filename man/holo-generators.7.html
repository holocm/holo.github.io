NAME

holo-generators - dynamically generate resource files for other plugins

DESCRIPTION

Generators are executable files placed under /usr/share/holo/generators. Holo will execute generators in lexical order before asking plugins to scan their resource files. An output directory will be supplied to each generator in the environment variable $OUT. Except for caching purposes (see below), generators shall only write to $OUT/.

The structure under $OUT/ is the same as for static resource files under /usr/share/holo/. For example, to generate resource files for Unknown node type L, a generator may place files at $OUT/files/.

Generated files placed under $OUT/ and static files under /usr/share/holo/ will be made available to plugins, with generated files taking precedence over static files. For instance, if a generator places a file at $OUT/files/20-webserver/etc/nginx/nginx.conf, the Unknown node type L plugin will see this file in its $HOLO_RESOURCE_DIR instead of /usr/share/holo/files/20-webserver/etc/nginx/nginx.conf.

ENVIRONMENT

Besides $OUT, generators are provided with the following environment variables.

Unknown node type over-text

Unknown node type L provides the user interface for this functionality.
